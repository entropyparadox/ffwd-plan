# 시스템 아키텍처

> "Technology is invisible when it works. The brand experience is what they remember."

## 설계 원칙: 기술은 브랜드 경험의 도구

시스템 아키텍처는 기술 문서가 아니라 브랜드 경험의 설계도다. 모든 서비스 레이어는 FFWD 헌장의 원칙을 기술적으로 실현한다.

| 브랜드 Augment | 기술 구현 |
|---|---|
| Shelf Pop — 첫인상에서 선택받는 힘 | 적응형 스트리밍 + 3초 내 로딩 |
| Language Bridge — 언어를 넘는 연결 | Whisper + DeepL 자동 자막 파이프라인 |
| Choice Architecture — 선택을 설계하는 힘 | AI 추천 + 전문가·대중 통합 랭킹 |
| Memory Marker — 기억에 남는 순간 | 실시간 투표 + 영화제 라이브 스트리밍 |
| Motion Signature — 브랜드의 움직임 | WebSocket 실시간 반응 + 푸시 알림 |

## 아키텍처 다이어그램

```
[Client Layer]
├── Web App (Next.js / React)
├── Mobile App (React Native)
└── Smart TV App (Year 2+)

[API Layer]
├── REST API (Node.js / Bun)
├── GraphQL (콘텐츠 쿼리)
└── WebSocket (실시간 투표/채팅)

[Service Layer]
├── Auth Service (OAuth2 + 소셜 로그인)
├── Content Service (업로드, 트랜스코딩, 자막)
├── Payment Service (Toss Payments)
├── Voting Service (전문가 + 대중)
├── Recommendation Service (AI 추천)
├── Notification Service (Push/Email/SMS)
└── Analytics Service (시청 데이터)

[Infra Layer]
├── Video: Cloudflare Stream / Mux
├── CDN: Cloudflare
├── DB: PlanetScale (MySQL) + Redis
├── Search: Meilisearch
├── Storage: Cloudflare R2
├── AI: Whisper (자막) + DeepL (번역)
└── Monitoring: Sentry + Grafana
```
